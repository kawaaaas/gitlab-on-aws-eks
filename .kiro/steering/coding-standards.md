# コーディング規約

## AWS CDK

- EKS v2 L2コンストラクト (`aws-cdk-lib/aws-eks`) を使用する
- スタック間の依存は明示的にpropsで渡す（暗黙的な依存を避ける）
- RemovalPolicy.DESTROY をデータを持つリソースに設定し、cdk destroy で確実に削除できるようにする
- コンストラクトIDは PascalCase で命名する
- マジックナンバーを避け、定数または cdk.json の context で管理する

## TypeScript

- strict モードを有効にする
- any 型の使用を避ける
- インターフェースでスタック間のprops型を定義する

## Helm Values

- k8s/values/ ディレクトリに各チャートの values.yaml を配置する
- デフォルト値からの変更点のみを記述し、変更理由をコメントで残す
- リソースリクエスト/リミットは学習用の最小値を設定する

## コメント規約（学習リソースとして）

- 各リソース定義に「なぜこのリソースが必要か」をコメントする
- EKS/K8s の概念が登場する箇所には簡潔な説明コメントを入れる
- GitLab コンポーネントの役割をコメントで説明する
- 「// NOTE:」で学習ポイントを、「// COST:」でコスト関連の注意を記す

## クリーンアップ

- 全スタックが cdk destroy で削除可能であることを保証する
- EBS ボリューム等の残留リソースに注意し、必要に応じて削除スクリプトを用意する
